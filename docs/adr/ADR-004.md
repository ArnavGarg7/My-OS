# ADR-004 — Proposal-first Mutations

- **Status:** Accepted (frozen, Sprint 5.5)
- **Date:** Phase 5.2

## Context

An AI that edits the user's plan directly is untrustworthy and hard to audit.

## Decision

Every AI-initiated data change is a **proposal**, never a direct write. The Chief/assistant returns a preview (summary + change lines + payload); the user accepts before anything is applied. Deterministic engines remain the only code that mutates state.

## Consequences

- The user is always in control; changes are reviewable and reversible before they happen.
- The assistant's conversational mutations ("move gym to Friday") surface as planner proposals.
- Enforced by the proposal shape in `@myos/ai/assistant` and the E2E validation scenarios.
