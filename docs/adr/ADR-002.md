# ADR-002 — Provider Abstraction

- **Status:** Accepted (frozen, Sprint 5.5)
- **Date:** Phase 5.1

## Context

My OS must not be locked to one AI vendor, and must work with no vendor at all (offline, CI).

## Decision

All providers implement one normalized interface. Model ids are **config** (`@myos/ai/config`), not code constants. Cloud providers take an **injected** client (real HTTP lives server-side, reading env keys); `@myos/ai` ships keyless. A capability→ordered-providers **Provider Policy** resolves which provider serves a request, always terminating in **Local**.

## Consequences

- Anthropic / OpenAI / Gemini / Groq / Local are fully interchangeable; no feature contains provider-specific logic.
- Adding a provider = one adapter + registry entry, no feature changes.
- Enforced by review; the reliability + benchmark suites treat providers uniformly.
