# ADR-007 — Tool Calling

- **Status:** Accepted (frozen, Sprint 5.5)
- **Date:** Phase 5.1 / 5.3

## Context

Grounded answers require the AI to fetch real data, not hallucinate it. Fetching must be safe and observable.

## Decision

The AI retrieves data through a **Tool Registry**. Each tool has a strict input/output schema, declared permissions, and telemetry. The pipeline is `lookup → permission check → input validation → execute`. The assistant's answer is composed **only** from executed tool results; if nothing grounds it, the Chief says the OS doesn't have that.

## Consequences

- No ungrounded factual claims; every citation traces to a tool result.
- Tool authorization is checkable ([ADR-009](ADR-009.md), security diagnostics).
- Server tool handlers bind the pure registry to real domain services — the seam that keeps `@myos/ai` keyless and DB-free.
