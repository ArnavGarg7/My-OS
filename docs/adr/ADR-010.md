# ADR-010 — Architecture Freeze

- **Status:** Accepted (Sprint 5.5)
- **Date:** Phase 5.5

## Context

Before Phase 6 (Autonomous Intelligence), the platform must be stable, understandable, and resistant to accidental architectural drift.

## Decision

Public interfaces are **frozen**. Modules may evolve internally, but the documented public APIs of Planner / AI / Knowledge / Timeline / Health / Resource / Notification / Automation are stable; breaking them requires a migration and a new ADR. Layering (shared → core → db → server → ai → app), single-barrel exports, contiguous migrations, package standards, and data classification are **enforced automatically** by the repository auditor in CI.

## Consequences

- New capabilities build on a stable foundation without accruing debt.
- Violations fail CI (`scripts/repository-audit.mjs`), not code review alone.
- The release candidate `v1.0.0-rc1` marks the frozen baseline.
