# ADR-001 — Deterministic Core

- **Status:** Accepted (frozen, Sprint 5.5)
- **Date:** Phase 2 onward

## Context

A life OS makes many calculations — scores, progress, forecasts, readiness, availability, recurrence. If these were spread across the UI, the server, or an AI model, results would be inconsistent and untestable.

## Decision

All business calculations live in `@myos/core` as **pure functions of their inputs** — no IO, no clock, no randomness. Anything time-dependent takes `now` as a parameter. The server layer supplies data and persistence; the AI layer never re-implements a calculation.

## Consequences

- Every rule is unit-testable in isolation and identical in dev/CI.
- The AI platform can trust the OS as the single source of truth ([ADR-004](ADR-004.md)).
- Determinism is enforced by review + the api-audit contract; a change to a derived value is a deliberate, testable act.
