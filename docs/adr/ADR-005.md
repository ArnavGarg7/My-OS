# ADR-005 — Offline-first Local Provider

- **Status:** Accepted (frozen, Sprint 5.5)
- **Date:** Phase 5.1

## Context

The platform must be fully functional with no API keys and no network — for development, CI, privacy, and resilience.

## Decision

The **Local provider** is a deterministic, offline implementation and the universal terminal fallback of the Provider Policy. Every AI capability produces a valid (if simpler) result on Local. Cloud providers are optional enhancements activated by env keys.

## Consequences

- Tests and CI run identically without secrets or network.
- Every reliability recovery path ends in Local — no failure leaves the user without an answer.
- Cost intelligence can quantify the savings Local delivers vs. a reference cloud model.
