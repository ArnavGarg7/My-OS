# ADR-009 — Telemetry & Observability Pipeline

- **Status:** Accepted (frozen, Sprint 5.5)
- **Date:** Phase 4.5 / 5.1 / 5.4

## Context

A production AI platform must answer, for any request: which provider, which tools, what context, how much cost, how long — and be reproducible.

## Decision

Two layers: (1) the tRPC `observed` middleware logs every procedure (module/operation/status/duration); (2) the AI pipeline emits a telemetry event per request and a complete **execution trace** (provider, prompt version, context builders, tool calls, memory, latencies, tokens, fallbacks). Traces support deterministic **replay**.

## Consequences

- Every AI decision is explainable and debuggable via the AI Developer Console.
- Cost, performance, and reliability are measured continuously against budgets.
- Enforced by the `@myos/ai` observability/performance/cost modules + `ai_execution_traces`.
