# Data Classification (Phase 4.5, Security)

Generated by `scripts/security-audit.mjs` from `apps/web/lib/security/classification.ts`. Every persisted domain carries a sensitivity class, and the AI-safe boundary states what may cross to an external model in Phase 5. **There is no AI in the system today — this is the contract Phase 5 must honour.**

## Levels

| Level | Meaning |
| --- | --- |
| `public` | Non-identifying, shareable as-is (enum labels, option lists, counts). |
| `internal` | Operational metadata, no personal content (ids, timestamps, statuses). |
| `sensitive` | Personal but not secret — reveals habits/health/finances if disclosed. |
| `private` | Secrets or free-text PII (auth tokens, email, journal/note bodies). |

## AI-safe boundary

Raw `sensitive`/`private` rows are **never** sent to an external model. Only the derived read models below — aggregated, scored, no free text or identifiers — are AI-safe:

- `summary`
- `signals`
- `statusSignal`
- `dashboard`
- `portfolio`
- `statistics`
